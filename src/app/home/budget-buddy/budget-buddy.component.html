Budget Buddy

A gamified web application for helping users manage their spending.
Users track spending, record income, recieve suggestions to help them stick to a budget, and earn rewards.

CHALLENGE

Budget Buddy was created and developed by myself and two other computer science students at Quinnipiac University. Built from the ground up, this web application uses a three-tier architecture to manage data, facilitate challenges, make recommendations, and help users manage their budget.
The development process was split up into several components:

  • Vision Statement
  • System Requirements
  • System Design
  • 8 Scrum Sprints
  • Final Report

PRE-DEVELOPMENT

VISION

We spent time drafting our ideas and creating our "vision" for what Budget Buddy would be.

We sought to build an application that reduced debt, incentivized staying up-to-date with personal finances, and was fun and engaging for users.

SYSTEM REQUIREMENTS & DESIGN

We organized our system requirements into different categories: the technology stack, user interface, and system features.


Technology Stack

Our tech stack utilizes a 3-tier architecture.

The client was built using the frontend framework, Angular, our server uses a Python Django framework, which communicates with our Postgre SQL database.

The client and server communicate via HTTP Requests using RESTFUL APIs

User Interface

The inital UI design was drafted in Adobe XD.

While the designs did not go through an extensive critique process, as the focus was on site development, we as a team did our best to follow UX best practices.

System Features

The following features were deemed most essential to the application, and were the foundation of the backlog and work units while constructing the prototype.

Prioritization of the work units helped the team stay on track and ensure that the core reqirements of the app were satisfied.

All reqirements were satisfied during the development process, except for mobile notifications, which were added to the backlog for future implementation.


RESULTS

Following 8 Scrum Sprints in the Agile development process, we successfully build a working prototype of Budget Buddy.

I acted as the team's primary frontend developer, working to implement the user interface in Angular. I set up the basic structure of the site, then moved on to create interactive components. These were either developed specifically for the project, or modified from the Prime NG component library.

UI Considerations

Notably, during Sprints 7 and 8, I focused primarily on cleaning up the existing UI. After receiving feedback from peers and advisors, as well as my own assessment of the current state of the project, I compiled a list of necessary UI updates that would improve the overall user experience, and help the app function overall.

Some of these changes included removing unnecessary or repetitive widgets, making data visualizations cleaner and more informative, establishing a consistent design system throughout the application, and condensing forms to reduce the time the user spends initalizing their budget.
