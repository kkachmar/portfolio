Platformer Game

A simple platformer game inherited and improved upon by my software development team. Under the guidance of a project manager, we worked together to improve the exsiting product.

CHALLENGE

This was a 2D platformer game initally created by another developer. As a simple platformer game, the user controlled a cat avatar that navigated around the map to reach a gold box at the end of the level. Our task was to use the software we inherited to repair bugs, add enhancements, and improve the game overall. I worked on a team comprised of four software developers under the direction of a product manager.

The development process was split up into several components:

  • Program Review and Assessment
  • Customer Requirements (User Testing)
  • Prioritizing Bugs and Enhancements
  • 4 Scrum Cycles
  • Final Report

USER TESTING


We conducted user testing to gain feedback on the game and to identify necessary bugs and enhancements. After the team and outside users tested the game, we gathered the following recommendations:

  • Fix game-breaking bugs
  • Provide instructions on how to play
  • Increase the difficulty level of the game
  • Create more interactive elements (enemies, something to collect, etc.)
  • Add a points system to keep the player engaged
  • Interactive obstacles (water, plants, etc)
  • Pause needs more functions, able to restart game, exit to main menu
  • The player should have a way to defeat enemies
  • Implement background music and sound effects
  • Add momentum-based movements
  • Increase level length and add more levels
  • Improve user interface (show lives, coins collected, enemies defeated, etc)

DEVELOPMENT PLAN

Once user testing was conducted, development plan was created and tasks prioritized based on their impact on gameplay and user requests.

Instructions for controls and option

Add instructions for controls and option to use WASD keys in addition to the arrowkeys

Ability to walk off the map

Prevent avatar from walking off the map and getting stuck off screen

Game Over Screen

Add game over screen when avatar falls off the map at the bottom of the screen

Interaction with water

Add feature so cat is harmed when it touches the water map tiles

Dinosaur Attack

Fix lagging dinosaur attack animation

Degree of Difficulty

Add additional levels of difficulty to make the game longer and more engaging

Ability to kill enemies

Add an attack feature so user can do more than simply avoid enemies

Pause Button

Only allow user to pause game during gameplay, not on menu screens

Overriding of Keys

Have most recent keystroke override previous keystrokes


IMPLEMENTATION

Below are the enhancements I personally worked on for this project. For the full project, please view our GitHub repository.


INTERACTION WITH WATER

I worked as part of a team to enhance the water map tile. The goal was to have it call the hurtPlayer() function that ultimately leads to a game over when the cat avatar falls into it.

In order to do so, our Player class had to recognize that the tile object it intersecting was labeled as our new TileType variable, "TileType.KILLER". If it was true that the tile was of type "KILLER," we then called the hurtPlayer() function utilized by enemies in the game.

ATTACK FEATURE

I worked as part of a team to add an attack feature to the game. Many users complained that there was no way for the cat to fight back against the enemies.

I decided to make a version of the Fireball class, FriendlyFire, and use this to implement a power-up that can be used to defeat enemies. Interacting with and instance of FriendlyFire caused an enemy to be removed from the map, rather than harming the player. We also developed a sprite of a "force field" that served as the enemy's attack.

In order to make the game more realistic, we created a boolean called hasPowerUp that was not set to true until the player came into contact with a purple flower map tile. Once hasPowerUp was true, pressing the space bar would generate an instance of the force field in the direction the player was facing, allowing the player to remove enemies from the map.
